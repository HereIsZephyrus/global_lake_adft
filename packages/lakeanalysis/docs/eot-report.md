## 1. 方法概述

### 1.1 EOT简介

EOT (Extreme Occurrence Tail) 是一种用于建模极端事件发生概率的统计方法。它假设极端事件服从广义帕累托分布（GPD），通过以下步骤识别极端事件：

1. **季节性去趋势**: 使用谐波回归（正弦/余弦）去除数据的季节性趋势
2. **残差标准化**: 将去趋势后的残差标准化
3. **极值建模**: 使用GPD建模极端事件的发生概率
4. **阈值选择**: 根据分位数阈值（如90%分位数）识别极端事件

### 1.2 模型结构

EOT模型的核心参数：

| 参数 | 含义 |
|------|------|
| β₀ | 截距（季节性均值） |
| β₁ | 线性趋势系数 |
| sin_1, cos_1 | 年度周期分量 |
| σ | 残差标准差 |
| ξ | GPD形状参数（尾部指数） |
| threshold | 极端事件阈值 |
| n_extremes | 检测到的极端事件数 |

### 1.3 数据范围

- **分析湖泊总数**: 1,198,396 个（每个tail独立分析）
- **分析类型**: 双尾分析
  - **High tail (高尾)**: 极端高水面积事件（洪涝）
  - **Low tail (低尾)**: 极端低水面积事件（干旱）
- **分位数阈值**: 0.90（10%和90%分位数作为极端边界）
- **时间跨度**: 2001年 - 2023年

## 6. 方法对比

### 6.1 三种方法对比

| 指标 | 月距平阈值法 | Hawkes过程法 | EOT极值法 |
|------|-------------|-------------|----------|
| 分析湖泊数 | 623,981 | 643,962 (收敛) | 1,198,396 |
| 收敛/成功率 | 100% | 63.76% | **100%** |
| 总事件数 | 1,853,952 | - | 15,689,466 |
| 平均事件/湖 | 2.97 | 12.53 | 6.55 |
| 中位事件/湖 | 2 | 10 | 6 |
| 方法类型 | 确定性阈值 | 统计推断 | 极值统计 |
| 捕捉特性 | 状态跳变 | 事件聚类 | 极端频率 |

### 6.2 主要差异

1. **月距平阈值法**: 基于简单的距平阈值，识别"旱涝急转"事件
2. **Hawkes过程法**: 捕捉事件的时间依赖性（传染效应），识别聚类特性
3. **EOT极值法**: 建模极端事件的频率分布，识别显著偏离正常的事件

### 6.3 EOT的优势

- **100%收敛率**: 对各种类型的湖泊数据都能成功建模
- **季节性建模**: 显式建模季节性周期
- **极端值建模**: 使用GPD建模尾部行为，更适合极端事件分析

## 7. 主要发现

1. **高收敛率**: EOT模型对全球湖泊达到100%收敛，优于Hawkes过程

2. **事件频率适中**: 平均每湖约6.5次极端事件，中位数为6次

3. **季节性明显**:
   - 洪涝事件集中在9月（夏末秋初）
   - 干旱事件集中在7-8月（盛夏）

4. **分布集中**: 约54%的湖泊经历5-9次极端事件

5. **2001年异常**: 该年同时是洪涝和干旱事件最多的年份

6. **事件总数巨大**: 共检测到约1569万次极端事件（高低尾合计）

## 8. 附录：数据库表结构

### 8.1 eot_results 表

```sql
-- 存储拟合的EOT模型参数
eot_results(
    hylak_id,              -- 湖泊ID
    tail,                  -- 'high' 或 'low'
    threshold_quantile,     -- 分位数阈值
    converged,             -- 是否收敛
    log_likelihood,        -- 对数似然值
    threshold,             -- 极端事件阈值
    n_extremes,            -- 极端事件数
    n_observations,        -- 观测数
    n_frozen_months,       -- 冻结月数（用于计算气候态）
    beta0, beta1,          -- 趋势参数
    sin_1, cos_1,          -- 季节性参数
    sigma,                 -- 残差标准差
    xi,                    -- GPD形状参数
    error_message,
    computed_at
)
```

### 8.2 eot_extremes 表

```sql
-- 存储检测到的极端事件
eot_extremes(
    hylak_id,              -- 湖泊ID
    tail,                  -- 'high' 或 'low'
    threshold_quantile,     -- 分位数阈值
    cluster_id,            -- 簇ID（事件编号）
    cluster_size,          -- 簇大小（通常为1）
    year, month,           -- 事件时间
    water_area,            -- 事件时的水面积
    threshold_at_event,     -- 事件阈值
    computed_at
)
```
